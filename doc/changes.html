IsaViz - Changes

Important Note

Several people have encountered difficulties importing local RDF files in IsaViz 1.2 (and might encounter the same problem if upgrading from 1.0 or 1.1 to 2.0 or 2.1): the model is not loaded, and the following error message is generated:

RDFErrorHandler.error: com.hp.hpl.jena.rdf.arp.MalformedURIException: No scheme found in URI.

The source of this error is a bad Defaut Base URI in your preferences (e.g. "online"). To correct this, select Edit/Preferences, and set the Default Base URI value to blank or to the well-formed URI of your choice.

version 2.1

  • Includes Jena 2.1 (up-to-date with respect to revised W3C RDF/XML Syntax Specification)
  • Improved usability under Mac OS X : command + mouse button now behaves as the right mouse button (requires Java 1.4.x or later)
  • Slight modifications to the GUI, for better integration with Mac OS X and the new J2SE 5.0 default look and feel
  • RDF resources in models exported as SVG documents now have an XLink href attribute whose value is their own URI
  • Requires a Java Virtual Machine 1.4.0 or higher (no longer compatible with Java 1.3.x)

version 2.0

  • Support for GSS (Graph Stylesheets), an RDF-based style language for rendering RDF models
  • Includes Jena 2.0 beta 1 (up-to-date with respect to last call W3C candidate syntax specification)
  • Support for RDF Datatypes
  • Better support for namespace prefix bindings (now extracted from the imported RDF files, and available in URI input text fields)
  • Import/Export plug-in interface featuring automatic detection (used by the Sesame plug-in)
  • Faster XML processing using Xerces 2.2
  • New navigation panel and keyboard shortcuts
  • Bug Fixes
  • Includes version 0.9.0 of the ZVTM library, which fixes a bug that made selecting nodes and edges in the graph difficult under certain circumstances (when the radar view was active)

version 1.2

  • Support for Jena 1.6.x (up-to-date with respect to last call W3C candidate syntax specification, except for datatypes which will be supported in IsaViz 2.0)
  • Support for Notation 3 import/export (through Jena)
  • Uses latest features of ZVTM 0.8.2 (enhanced magnification capabilities, fonts, etc.)
  • Better support for internationalization (i18n) provided input files use the UTF-8 charset (screenshot)
  • Customizable fonts and background color
  • Printing capabilities
  • A radar window to get an overview of the graph
  • Added a back feature to return back to the previous location in the graph (multi-levels, available in the View menu or using keyboard shortcut Ctrl+B)
  • User-changeable error mode (default, strict, lax)
  • Modified makefiles and launch scripts to reflect the change of name XVTM -> ZVTM
  • Bug fixes

version 1.1

  • Includes new version of XVTM : the mouse cursor has been partially decoupled from Views and is therefore not always dependent on the view's refresh rate (which itself depends on the complexity of the graph). This means that it should make controlling the mouse cursor in large graphs much easier.
  • Fixed a bug in the SVG Export module (vertical offset problem in some cases)
  • Fixed the "looping spline" problem related to graphviz not taking into account the direction of the spline when encoding the SVG path coordinates
  • Modified layout of items in the list of properties (property browser)
  • Modified makefiles and launch scripts to reflect the change of name VTM -> XVTM
  • Fixed temporary file deletion bug (temp files were not deleted when exiting IsaViz)
  • Updated RDFLoader to take into account changes in Jena 1.3.2 w.r.t separation between namespace and local name of a resource

version 1.0a

  • Adapted the dot/SVG module to support the new SVG output structure in Graphviz/dot 1.8.4

version 1.0

  • First public version
